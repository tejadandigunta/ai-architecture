Inside an LLM

You hit send. Three seconds later you get an answer. Here is what actually happened.

What happens inside an LLM?
01

Tokenization

Your text gets broken into tokens, not words. The model never sees letters or words, only numbers.

Un believ able = 3 tokens
Token IDUn -> 1428 Token IDbeliev -> 29391 Token IDable -> 481
02

Embeddings

Each token becomes a vector. Similar meanings are stored close together in the model's vector space.

Royalty cluster Food cluster
Vector DB: meaning group A King [0.82, 0.14, 0.71] Queen [0.80, 0.18, 0.75] Prince [0.77, 0.11, 0.69] Crown [0.74, 0.16, 0.72]
Vector DB: meaning group B Pizza [0.12, 0.91, 0.23] Burger [0.18, 0.88, 0.20] Pasta [0.10, 0.84, 0.27]
03

Attention

The model asks: "Which words explain this word?" For bank, river and steep get high attention, so bank means river edge.
